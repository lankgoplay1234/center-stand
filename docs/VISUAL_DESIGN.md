# Center Stand Visual Design Guide

## 방향

- 캐릭터는 선택 화면과 인게임에서 같은 픽셀 아트 자산을 사용한다.
- 무기, 실루엣, 역할 색만 보고도 여섯 캐릭터를 구분할 수 있어야 한다.
- 전투 배경은 중앙을 어둡고 단순하게 유지하고, 테마 정보는 가장자리와 낮은 채도의 발광으로 전달한다.
- 적, 투사체, 피해 숫자와 HUD는 배경보다 높은 명도와 채도를 유지한다.
- 전투 효과는 짧고 강하게 표현하되 풀링된 객체 수를 늘리지 않는다.

## 자산 규격

- 캐릭터: 투명 PNG, 128×128, 최근접 필터, 권장 파일 크기 25KB 이하
- 스테이지 배경: PNG, 512×512, 권장 파일 크기 500KB 이하
- 캐릭터 키와 경로, 스테이지 테마 구간은 `VisualAssetData.ts`에서만 관리한다.
- 배경 테마는 1, 21, 41, 61, 81스테이지에서 전환한다.

## 전투 피드백

- 일반 피해: 밝은 크림색 숫자가 작은 좌우 편차와 함께 빠르게 확대·상승한다.
- 치명타: 금색 `CRIT!` 문구, 큰 확대, 회전 편차, 폭발과 원형 파동을 함께 사용한다.
- 치명타 기본 확률과 배율은 `CriticalHitSystem.ts`에서 관리한다.
- 캐릭터 스프라이트의 반동·검격 돌진·마법 부유 동작은 기존 공격 판정과 독립적으로 실행한다.

## 후속 디자인 권장 사항

- 일반 적과 대장 적도 역할별 저해상도 스프라이트 시트로 교체하되 현재 풀 객체 수를 유지한다.
- 캐릭터마다 대기 2프레임, 공격 2~4프레임의 작은 스프라이트 시트를 추가한다.
- 체력 위험, 대장 등장, 99레벨 달성에 짧은 테두리·화면 가장자리 연출을 추가한다.
- 실제 Android/iOS 기기에서 배경 대비, 잔상, 발열과 치명타 숫자 겹침을 최종 조정한다.
