## 알려진 문제

- `RANK-002` UI와 클라이언트 테스트는 통과했지만 실제 API 배포 주소와 서버 발급 완주 proof가 없어 공개 기록 업로드는 비활성화되며 최종 검수는 `BLOCKED`다.
- `PERF-002` Pixel 5 에뮬레이션 10분·120표본은 완료됐지만 실제 Android/iOS 기기의 발열·배터리·GPU 스로틀링 측정이 없어 최종 검수는 `BLOCKED`다.
- `AUDIO-001` 오디오 상태와 desktop/mobile 이벤트 검사는 통과했지만, 실제 기기의 음량·음색·반복 체감 청취가 필요해 최종 검수는 `BLOCKED`다.
- `BALANCE-004` 24회 결정론적 시뮬레이션은 전 캐릭터 목표 범위와 편차를 충족하지만, 완료 조건의 실제 40분 플레이 사망 횟수 표본은 사람의 수동 플레이가 필요해 최종 검수는 `BLOCKED`다.
- `BALANCE-002` 자동 계산은 기본 39분·50회 부활 포함 40분을 충족하고 시간 가속 브라우저 검사도 통과했으나, 완료 조건의 실제 40분 플레이 표본은 사람의 수동 플레이가 필요해 최종 검수는 `BLOCKED`다.
- 기본 BGM과 업그레이드음은 브라우저 합성음이며 실제 모바일 기기의 음량·음색 수동 검수가 필요하다.
- 100 적 테스트 버튼과 디버그 HUD는 개발 모드에서만 표시된다.
- `PERF-003` 최종 하위 10% FPS가 27.99로 30 FPS 목표에 미달해 BLOCKED 상태다. 실제 기기 프로파일링 또는 렌더링 정책 결정이 필요하다.
- 실제 Android/iOS 기기의 발열·배터리·GPU 스로틀링은 아직 측정하지 않았다.
- 300개 이동 적에서는 공간 인덱스 재구축 비용이 커 현재 bounded selection을 유지한다.
- Playwright는 로컬 Chromium 설치가 필요하며 최초 실행 시 `npx playwright install chromium`을 실행해야 한다.
- 바스티온·룬 메이지·니들 스트라이커·스톰 컨덕터의 특수 강화는 기존 공통 사거리·효과 범위 증가를 유지한다.
- 넉백은 현재 즉시 위치 이동 방식이며 향후 애니메이션 보간을 추가로 다듬을 수 있다.
- 온라인 랭킹의 API 계약·보안 정책·UI는 구현됐지만 production 저장소 어댑터, 서버 발급 proof와 배포 환경이 필요하다.
- 일반 사망은 결과 화면으로 이동하지 않으며 기존 GameOverScene은 100스테이지 완주 결과 화면 전환용으로 보존되어 있다.
- 현재 밸런스는 데이터·계산 테스트 기반의 1차 조정이며, 장시간 실제 플레이 지표에 따른 추가 튜닝이 필요하다.
- 40분 완주 시간은 자동 계산과 시간 가속 브라우저 검사 기준이며, 실제 1회 완주 플레이 표본은 아직 수집하지 않았다.
- 400회 완주 기준은 역할별 대표 업그레이드 분배와 58% 처치율 시뮬레이션 기준이며, 실제 구매 패턴·장시간 플레이 표본은 아직 수집하지 않았다.
- 역할별 분배 평균 사망 횟수 40.96~51.83회는 결정론적 전투 모델 결과이며, 실제 플레이어의 선택과 적 배치에 따른 40분 완주 표본 검증이 필요하다.
- 공격 모션은 외부 이미지 없이 Phaser 도형으로 구성한 1차 연출이며 실제 모바일 기기의 잔상·색상·고주사율 체감 검수가 필요하다.
- 대장 몬스터가 포함된 실제 10분 모바일 성능과 후반 40분 완주 난이도는 실제 기기에서 추가 측정해야 한다.
- 최종 승급 외형은 작은 모바일 화면에서 장식 간격과 금빛 테두리 가독성을 실제 기기로 추가 검수해야 한다.
- 신규 픽셀 자산은 에뮬레이션과 인앱 브라우저에서 확인했으며 실제 고밀도 Android/iOS 화면의 크기·대비는 추가 수동 검수가 필요하다.
- 치명타는 모든 캐릭터에 공통 8%·1.75배로 적용한 1차 값이며 장시간 완주 밸런스 표본에 따라 캐릭터별 조정이 필요할 수 있다.
- 몬스터는 기존 고성능 도형 렌더링을 유지한다. 역할별 픽셀 몬스터와 프레임 애니메이션은 후속 디자인 작업이다.
