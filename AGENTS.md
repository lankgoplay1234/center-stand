# AI Agent Working Agreement

이 저장소의 에이전트는 안정적으로 검증 가능한 작은 변경을 누적한다.

## 필수 작업 규칙

1. 한 번에 하나의 작업만 수행한다.
2. 작업 전 `PROJECT_STATUS.md`, 해당 TASK 항목, 관련 코드와 데이터/타입 파일을 읽는다.
3. 기존 기능을 사용자 승인 없이 제거하거나 동작을 바꾸지 않는다.
4. 변경 범위를 완료 조건에 필요한 최소 범위로 제한한다.
5. 기능 추가 전 `src/game/types`, `src/game/data`의 기존 정의를 우선 확인한다.
6. 게임 수치와 비용 공식을 씬에 하드코딩하지 않는다.
7. 비밀키, 토큰, 개인 식별 정보를 코드·문서·로그에 작성하지 않는다.
8. 씬 통합, 상태 관리 교체, 디렉터리 대이동 등 중요한 구조 변경은 사용자 승인 없이 진행하지 않는다.
9. 적·투사체·이펙트의 대량 처리 경로에서 프레임별 객체/타이머 생성을 피한다.
10. 모든 변경 후 `npm run typecheck`, `npm run lint`, `npm run test`, `npm run build`를 실행한다.
11. 빌드 실패 상태를 완료로 처리하지 않는다. 테스트 실패 시 원인과 재현 명령을 기록하고 작업을 중단한다.
12. 완료 후 `PROJECT_STATUS.md`와 `CHANGELOG.md`를 갱신한다.
13. main 브랜치에 직접 대규모 변경하지 않는 워크플로를 전제로 하며, 작업 브랜치와 작은 커밋을 선호한다.

## 작업 흐름

1. `TASKS.md`에서 상태가 `READY`인 작업 하나를 선택한다.
2. 상태를 `IN_PROGRESS`로 바꾸고 완료 조건과 관련 파일을 확인한다.
3. 구현과 단위 테스트를 함께 작성한다.
4. 네 가지 검증 명령을 모두 통과시킨다.
5. 상태를 `REVIEW`로 바꾸고 변경 파일, 테스트 결과, 알려진 제약을 기록한다.
6. 검토 완료 뒤에만 `DONE`으로 변경한다.

## 상태 값

`BACKLOG`, `READY`, `IN_PROGRESS`, `BLOCKED`, `REVIEW`, `DONE`만 사용한다.

## 텔레그램 준비

보고 구현은 `scripts/telegram/TelegramReporter.ts`의 인터페이스를 따른다. `.env` 값은 출력하지 않으며, 실제 전송은 별도 승인된 작업에서만 구현한다.
